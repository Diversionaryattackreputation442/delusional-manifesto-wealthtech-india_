# Contributing to The Delusional Manifesto

Thanks for wanting to contribute. The only rule: **no incrementalism.**

---

## The Rule

If your idea is "what if we added AI to [existing product]," it doesn't belong here.

If your idea is "what if [fundamental assumption about money] is wrong," pull up a chair.

---

## What You Can Contribute

**New chapters** — A chapter is a first-principles argument, not a feature list. It should make someone uncomfortable or excited (ideally both). See the existing chapters for tone and structure.

**Architecture documents** — Concrete technical documents that answer "how would this actually work?" with real APIs, real constraints, real workarounds. No hand-waving.

**Product specs** — Full standalone specs like the ones in `product-specs/`. They should include: the problem, the mechanism, what it doesn't do, the economics, and a trust/ethics section.

**Prototype extensions** — New screens, new interactions, new data visualizations that fit the organism mental model. React only. No external dependencies beyond what's already in the prototype.

**Challenges** — Open an issue that challenges a core assumption in an existing chapter. Frame it as a question, not a rebuttal. The best challenges will be incorporated into the documents themselves.

---

## What Doesn't Belong

- Incremental improvements to existing financial products
- "AI-powered [thing that already exists]"
- Comparisons to US/EU fintech that ignore India Stack
- Anything that assumes the user needs to learn more to benefit

---

## How to Submit

1. Fork the repo
2. Create a branch: `git checkout -b your-idea-name`
3. Write your contribution — follow the style of existing documents (direct, opinionated, specific)
4. Open a pull request with a one-line summary of the central claim

PRs without a clear central claim won't be merged. The manifesto is a collection of arguments, not a collection of information.

---

## Style Guide

- **Direct voice**: "The system does X" not "the system could potentially do X"
- **Specific**: Name the API, name the platform, name the rupee amount
- **Opinionated**: State what you believe and why, not just what's possible
- **No buzzwords**: "AI-powered," "seamless," "frictionless," "ecosystem" — avoid these unless you're defining exactly what you mean
- **India-first**: Default to Indian context (INR, Indian platforms, Indian regulations). The global angle is secondary.

---

## On the Word "Delusional"

It's intentional. The manifesto is written from the perspective that the current state of Indian wealthtech is inadequate and that the right response is to imagine something radically different — not to optimize the existing model. Contributions should share this orientation.

"Pragmatic" contributions are welcome elsewhere. Here, we're allowed to be unreasonably ambitious.
